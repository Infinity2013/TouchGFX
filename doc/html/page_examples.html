Examples

Custom Transition Example

The Custom Transition example shows how to implement custom transitions and how these are used. The visual effect of the transition implemented in this example, resembles two doors being closed and opened from both sides of the screen. Both doors closes to hide the screen the transition is coming from. The doors then open again to reveal the screen the transition is going to. The doors are implemented by moving Images across the screen. A SnapshotWidget is used to capture a snapshot of the screen it transitions away from, which is used when the doors are closing.

Drag Example

The Drag Example illustrates a way to create draggable elements. This is, in this case, done by mixing the Snapper mix-in into a Button. The snapper will set the button draggable and receive callbacks when the button is dragged. When clicked the snapper will display a highlighted image to give visual feedback that a drag is taking place. When the drag is ended the snapper receives another callback and can decide whether to snap to a new position or not, displaying the appropriate element at the appropriate location. The Drag Example comes in both a 16 and a 24 bit version.

The element is dragged over the drop zone

Keyboard Example

The Keyboard Example shows how to implement an on screen keyboard by using the Keyboard widget. The keyboard implemented in this example, has both alpha and numeric keys, as well as a backspace key, a space key and a caps lock key.

The example covers how keyboard layouts are created and used to define the visual appearance of the keyboard. Furthermore, it covers how Keyboard mappings are set up, and how these can be changed at runtime to change the keys displayed on the keyboard. The implementation of custom keys, such as caps lock and backspace are also covered in the example.

Text Example

The Text Example shows how to use and display texts. Texts are specified in the assets/texts/texts.xlsx file. These texts along with the fonts in the assets/fonts directory are automatically transformed into resources in the generated directory.

Texts have a font, size, alignment and color. Texts can be either static or dynamic. Dynamic texts have wildcards corresponding to %s in regular C++ strings.

The example shows a static text and a dynamic text.

Text Rotate Example

The Text Rotate Example shows how to write text in one of four directions. Text is normally written left to right, being rotated 0 degrees, but it is possible to rotate the text by 90, 180 or 270 degrees at runtime, which implies that the text direction is not specified in the assets/texts/texts.xlsx file, but is set using a function call at runtime.

Transition Example

The Transition Example shows how to use transitions. Transitions are the mechanism used for changing between screens.

The example uses SlideTransition. The SlideTransition is one of the built in transitions.

The transition will grab a snapshot of the screen initially and then perform a soft horizontal or vertical slide.

In the example, the snapshot slides off to one side while the new screen slides in.
