Development Environment

The easiest way to setup a TouchGFX development environment is to use one of the few concrete methods supplied by TouchGFX, as explained below. The guidelines in Custom detail the tool-chain required to setup a custom environment, on Linux, for instance.

TouchGFX MinGW Visual Studio Custom Environment
TouchGFX ships with a simple MSYS/MinGW shell with the required tools and compilers installed and configured for compiling on Windows. Extract the contents from the self-extracting zip to the root of your C: drive, and you are ready to start building and running applications (See Build and Execute). TouchGFX supplies solutions and libraries for Visual Studio 2010, 2012, and 2013. If you wish to use either version to develop TouchGFX applications see Visual Studio and Visual Studio Project Creation for further details. If you use Linux, or for any reason wish to setup your own development environment, the tools you need to install are:
  1. gcc 4.9.1+
  2. make 3.8.1+
  3. ruby 1.9.x
  4. roo (Excel interface Ruby gem)
  5. GCC compiler for ARM processors (See GCC Building applications with GNU GCC compiler for ARM) 4.9.3
Warning
If using the TouchGFX MinGW development environment for Windows, avoid using spaces in any of the foldernames in your project as this may caused undefined MinGW behavior. This warning is also issued by the MinGW shell upon startup.
Note
Linux - libraries only provided for 64-bit systems.
