Demos

This page describes the demos included in the TouchGFX distribution located under app/demo/. The demos will run on both PC simulator (Windows and Linux) and on selected demo boards that support the demos resolution. Source code for all demos can be found in the app/demo/ folder, for each application respectively.

Compiling and running

The demos all support Make and Visual Studio build for the PC simulator and IAR, Keil or GCC builds for selected evaluation boards. This is done in the usual way, see Build and Execute for more information.

Demo 2015 (5.7" - 640x480)

The TouchGFX demo 2015 5.7" (app/demo/touchgfx_demo2015) is a quite extensive demo showing some of the graphical possibilities that TouchGFX offers.

The full source code and all image files are included in the release and should serve as inspiration and as an introduction on how to make complex graphical elements and views. The demo has a carousel menu that lets you choose between five separate independent demos. The carousel menu can be changed to a menu structure with animating boxes.

Note
Since the focus of this demo is to demonstrate how to make complex graphics, most of the screen do not use data from the model class, as would a real application. Instead, they mostly use hard-coded values as input.

Demo 2015 (4.3" - 480x272)

The TouchGFX demo 2015 4.3"(app/demo/touchgfx_demo2015_480x272_8MB) shows some of the graphical possibilities that TouchGFX offers while running on very small devices. This demo is designed to run on the lower range of Cortex M3/M4 e.g. 120 MHz NXP LPC4088.

The demo consist of a full screen menu that the user can swipe through and choose one of the small independent demos.

Demo 2014 (4.3" - 480x272)

The TouchGFX demo 2014 (app/demo/touchgfx_demo2014_480x272) is a quite extensive demo showing some of the graphical possibilities that TouchGFX offers.

The full source code and all image files are included in the release and should serve as inspiration and as an introduction on how to make complex graphical elements and views. The demo is divided into many small screens that each contain a separate independent demo. Each demo can be selected from the main page, which contains a scrollable menu.

Note
Since the focus of this demo is to demonstrate how to make complex graphics, most of the screen do not use data from the model class, as would a real application. Instead, they mostly use hard-coded values as input. One exception to this is the HomeAutomation screen. The HomeAutomationView uses the presenter and model in the correct way. The data is, of course, still hardcoded (in the model) since the demo naturally does not have any real input or output.

Demo 2014 small (2.4" - 240x320)

The TouchGFX demo 2014 small (app/demo/touchgfx_demo2014_240x320) is a reduces version of demo 2014. It is custom made for the ST STM-32F429 Discovery board. The resolution is QVGS (320x240) and only internal flash is used for storing image date (2 MB).

The menu screen (Demo 2014 small)

One part of the demo shows the effect of TouchGFX in combination with the ST Chrom Art acceleration of the STM32F429 MCU. The result is a very low MCU load while performing advanced alpha blended animations.

The ST chrom art screen (Demo 2014 small)

Home Control Demo (7.0" - 800x480)

The Home Control Demo is a 7.0" demo designed to show some of the capabilities of TouchGFX on a large display. Contrary to other demos described on this page, this demo uses the Model class, as it was intended, to store state between screens.
